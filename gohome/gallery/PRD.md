> 컨셉 한 줄에서 만든 초안(0.1)에 대표·기획 결정 12건을 반영한 0.2입니다. 핵심 규칙(소멸·편성·리스펙트·공유·댓글·관람권한·재생 모델)이 정해져 상태를 Review로 올립니다. 남은 것은 **화면 자료가 필요한 3건**(예고편·빈 시간·오류 화면 — 제안 반영, 디자인 확정 대기)과 **법무 검토**뿐입니다(§12·`decisions.md`).

> **결정에서 도출된 운영 모델 — "라이브 동시 방영"**: 한 시간 동안 모두가 같은 작품을 같은 시간 축으로 보고(중간 진입은 현재 지점부터), 개별 일시정지·탐색은 없으며, 끝나면 작품은 소멸하고 작가 프로필에 기록만 남깁니다.

## 1. 기능 한 줄 정의

갤러리는 Cre8orClub의 두 번째 메뉴로, 숏폼의 반대인 롱폼입니다. 한 시간에 단 하나의 작품만, 유틸 메뉴 없이, 모두가 같은 시간 축으로 함께 보는 시간제 라이브 전시 공간입니다.

## 2. 사용자와 문제

- 주요 사용자: 크리에이터의 긴 호흡 작품(포트폴리오)을 몰입해서 보고 싶은 관람객, 그리고 한 시간 동안 한 명에게 무대를 받는 크리에이터
- 사용자가 겪는 문제: 숏폼 피드에서는 긴 작품이 끝까지 보이지 않고, 무한 스크롤·유틸 버튼이 몰입을 끊습니다
- 이 기능이 해결하는 것: 한 시간에 한 작품만 남겨 선택 피로를 없애고, 화면에서 군더더기를 걷어내 작품에 집중하게 하며, "지금 아니면 사라진다"는 라이브 동시성으로 함께 보는 경험을 만듭니다

## 3. 목표

- 제품 목표: 숏폼과 구별되는 '한 시간 한 작품' 라이브 몰입 경험을 갤러리 메뉴로 만든다
- 사용자 목표: 지금 이 시간의 단 하나의 작품을 방해 없이 끝까지 본다
- 완료 후 기대되는 상태: 관람객이 알림 → 메뉴 → 작품 → 메뉴(리스펙트·공유·댓글) → 다음 예고까지 끊김 없이 돈다
- (성공 판단 수치는 아직 정하지 않아 단정하지 않습니다 — §12)

## 4. 범위

### 포함(이번에)

- 갤러리 진입 화면 — 이번 한 시간의 작품 카피 + 1시간 역카운트다운
- 작품 재생 — 유틸 메뉴 없이, 개별 조작 없이 작품만(라이브)
- 풀화면 메뉴 — 리스펙트·공유·댓글 + 닫기 (작품은 뒤에서 계속 재생)
- 댓글(v1 포함) — 모더레이션·신고 동선 포함
- 닫기 → 다음 시간 예고편
- 한 시간 한 작품 편성(운영자 선발) + 정시 교체 + 종료 시 소멸
- **방영 알림**(곧 시작·방영 중)

### 제외(이번엔 안 함)

- 추천·피드·검색 등 다른 메뉴 연계 (이 PRD 범위 밖)
- 지난 작품 다시보기·아카이브 (소멸 모델 — 작가 프로필의 출품 기록만 남김)
- 한 시간 단위 외의 편성(주간·상시) — 컨셉에 없음
- 작품 업로드·등록 흐름 (별도 기능)
- 작품 선발·편성 운영 도구(어드민) — 별도 기능, 이 PRD는 노출 규칙까지

## 5. 진입점과 화면 흐름

진입점은 앱 두 번째 메뉴 "갤러리"와 **방영 알림**입니다. 컨셉의 핵심은 **탭이 거듭될수록 화면이 단계적으로 열리는 것**입니다(카피 → 작품 → 메뉴).

- 진입 화면: 갤러리 진입(작품 카피 + 역카운트)
- 주요 화면: 작품 재생(라이브) → 풀화면 메뉴(오버레이)
- 다음 화면: 다음 시간 예고편 → (다시) 갤러리 진입
- 끊기면 안 되는 흐름: 진입 → 재생 → 메뉴 → 닫기 → 예고

### S1. 갤러리 진입

- 정상: "한 시간, 단 하나의 작품" 카피와 남은 시간 역카운트다운을 보여준다. 비로그인도 관람 진입 가능. 탭하면 작품 재생으로(중간 진입이면 현재 방영 지점부터).
- 빈 상태(편성 없음): "다음 전시까지" 카운트다운 + 다음 예고편 + 추천 크리에이터 입구 *(제안 — 디자인 확정 대기, I-2)*
- 실패: 시간/편성 정보를 못 불러옴 → 재시도 안내
- 다음: 작품 재생

### S2. 작품 재생 (라이브)

- 정상: 다시 탭하면 어떤 유틸 메뉴도 없이 작품만 재생한다. **개별 일시정지·탐색은 없다**(모두 같은 시간 축).
- 실패: 재생 오류·네트워크 끊김 → 자동 재시도, "방영은 계속됩니다 — 복구되면 현재 지점부터 이어서" 안내(시간은 흐름) *(제안 — 디자인 확정 대기, I-3)*
- 다음: 풀화면 메뉴(작품을 1회 탭)

### S3. 풀화면 메뉴 (오버레이)

- 정상: 작품을 탭하면 리스펙트·공유·댓글 메뉴와 닫기 버튼이 작품 위에 **오버레이**로 뜬다. **메뉴가 떠 있어도 작품(라이브)은 멈추지 않고 계속 재생된다**(C-1). 작성 행위(리스펙트·댓글)는 로그인 필요.
- 시간 종료: 메뉴/재생 중 한 시간이 끝나면 작품은 소멸하고 다음 시간 예고편으로 전환된다.
- 다음: 작품 재생(메뉴 닫음) 또는 다음 시간 예고편(닫기 버튼)

### S4. 다음 시간 예고편

- 정상: 닫기를 누르면 다음 한 시간에 나올 작품 예고편(15~30초 영상 트레일러 + 작가명 + 시작 시각)을 보여준다 *(제안 — 형식 디자인 확정 대기, I-1)*.
- 빈 상태: 다음 편성이 없으면 "다음 전시 준비 중" + 카운트다운
- 다음: 갤러리 진입

## 6. 핵심 사용자 시나리오

1. 관람객이 '곧 시작' 알림을 받고 갤러리로 들어온다 → 이번 시간 작품 카피와 역카운트를 본다.
2. 다시 탭한다 → 유틸 없이 작품만 본다. 중간에 들어왔다면 지금 방영 중인 지점부터 함께 본다.
3. 작품을 탭한다 → 작품 위에 리스펙트·공유·댓글 메뉴가 뜨고, 작품은 뒤에서 계속 재생된다. 리스펙트를 보낸다(로그인, 1인 1회).
4. 닫기를 누른다 → 다음 시간 예고편을 본다.
5. 한 시간이 끝나면 작품은 내려가고, 작가 프로필에 출품 기록만 남는다.

| FR-004 | 로그인 관람객이 작품에 리스펙트를 1인 1회 보내고 취소할 수 있다 | 로그인 사용자가 리스펙트를 누르면 1회 반영·표시되고, 다시 누르면 취소된다. 비로그인은 로그인 유도 | F-004 |
| FR-005 | 관람객이 작품을 외부로 공유할 수 있고, 만료된 링크는 안내 후 작가 프로필로 보낸다 | 공유 시 공유 수단이 뜨고, 소멸한 작품 링크를 열면 만료 안내 화면을 거쳐 작가 프로필로 착지한다 | F-005 |
| FR-006 | 로그인 관람객이 댓글을 남기고, 부적절 댓글을 신고할 수 있다(모더레이션 포함) | 로그인 사용자가 댓글을 남기면 표시되고, 신고하면 접수되어 모더레이션 큐로 들어간다 | F-006 |
| FR-007 | 닫기를 누르면 다음 시간 예고편으로 전환한다 | 닫기 시 다음 시간 예고편 화면으로 이동한다 | F-007 |
| FR-008 | 다음 한 시간에 나올 작품 예고편을 보여준다 | 예고편 화면에 다음 작품 트레일러·작가명·시작 시각이 표시된다(영상 트레일러 제안, 형식 확정 대기) | F-008 |
| FR-009 | 운영자가 선발한 단 하나의 작품을 한 시간마다 노출하고, 정시에 교체하며, 종료된 작품은 소멸시킨다 | 같은 시간대에는 항상 한 작품만 보이고, 정시에 다음 작품으로 바뀌며, 지난 작품은 내려가고 작가 프로필에 기록만 남는다. 작품 길이는 60분 이내 | F-009 |
| FR-010 | 곧 시작·방영 중 작품을 사용자에게 알린다 | '곧 시작'과 '방영 중' 시점에 알림이 발송되고, 알림을 누르면 갤러리로 진입한다 | F-010 |

## 9. 상태와 예외

| 상황 | 처리 | 중요도 |
|---|---|---|
| 기본 상태 | 진입 시 작품 카피 + 역카운트, 탭 시 라이브 재생 | 꼭 |
| 로딩 상태 | 작품·시간 불러오는 동안 로딩 표시(역카운트는 유지) | 꼭 |
| 빈 상태 | 편성 없음 → "다음 전시까지" 카운트다운 + 예고편 + 추천 *(제안, I-2)* | 꼭 |
| 실패 상태 | 재생 오류·끊김 → 자동 재시도 + 현재 지점부터 이어서, 시간은 흐름 *(제안, I-3)* | 꼭 |
| 권한 없음 | 비로그인이 리스펙트·공유·댓글 작성 시 → 로그인 유도(관람은 그대로) | 꼭 |
| 중복 요청 | 리스펙트 중복 탭 → 1인 1회로 멱등 처리(두 번째는 취소로 동작) | 곧 |
| 시간 종료 | 메뉴/재생 중 종료 → 작품 소멸, 다음 시간 예고편으로 전환 | 꼭 |
| 만료 링크 | 소멸 작품의 공유 링크 열람 → 만료 안내 후 작가 프로필 | 곧 |

- 불변식(항상 참): 같은 시간대에는 언제 들어와도 단 하나의 작품만, 같은 방영 지점으로 보인다.
