고스트프로토콜 — 작업 메커니즘

어떤 입력이든 받아 PRD와 기능정의 리스트로 바꿔 개발자에게 그대로 넘기는 기획 핸드오프 스킬. 절차·분기·반복을 하나도 생략하지 않았습니다.

색 막대만 따라가면 핵심 흐름(0 → A → B → C → D)이 드러납니다.
회색 줄까지 보면 단계별 세부 절차가 빠짐없이 보입니다.
갈림길·↔·오른쪽 루프 레일이 분기·연결·반복의 실제 흐름을 보여줍니다.
0 메뉴 기준선→ A 입력→ B 변환→ C 7파일 패키지→ D 검증·배포↺ ↺ 반복

BPRD·기능정의로 변환Phase 2~5
  • 제품·기능 모델 정리 — 목적 · 사용자 · 핵심 객체·행동 · 성공 상태. 본질을 한 줄로
  • 기능 추출(15컬럼) — 화면 있으면 역추출, 컨셉만이면 구조화. 행 1개 = 행동 1개
  • 정책·예외·상태 점검 — 레드플래그 → 정책·예외·경계·상태 4갈래 → 컴플라이언스·IP(법무 라우팅) → 시스템 파급
  • PRD 12섹션 작성 — 근거 없는 정책·예외는 확정 않고 decisions·미결정으로
  • 4가지를 절대 안 섞음 — 근거(화면) · 의도(컨셉) · 제안(AI) · 미결정(사람)
